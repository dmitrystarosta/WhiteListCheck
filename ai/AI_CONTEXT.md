## Назначение проекта

Android-приложение **«Белый список?»** помогает пользователю в РФ понять,
что происходит с его интернетом: включён ли в регионе режим «белого списка»
мобильного интернета (перечень доступных сайтов, который ведёт Минцифры),
пропал ли интернет целиком, работает ли VPN — или всё в норме.

Проект создан пользователем-новичком (Дмитрий / Dmitry Starosta,
GitHub: dmitrystarosta) полностью с помощью ИИ. Это его первое приложение.

## Краткое описание приложения

По нажатию кнопки «Проверить» приложение параллельно опрашивает три группы
сайтов лёгкими HTTPS HEAD-запросами (к favicon.ico, таймаут 4 с):

| Группа | Смысл | Состав (встроенный список) |
|---|---|---|
| A — белый список | эталон доступности | Яндекс, ВКонтакте, Госуслуги, Mail.ru |
| B — обычный интернет (вне списка) | контроль | Habr, 4PDA, Google, Википедия |
| C — заблокированные в РФ | контрольная группа | Instagram*, X (Twitter), Rutracker |

Вердикт по большинству в группе:
- A работает, B нет → «включён белый список»
- A и B работают, C нет → «всё в норме»
- Работает всё, включая C → «включён VPN или вы вне РФ»
- Не работает ничего → «интернет недоступен вообще»
- A нет, B работает → UNKNOWN «непонятная ситуация» (доработка
  запланирована, см. TODO)

## Основные функции

- Ручная проверка одной кнопкой; сворачиваемые карточки групп со сводкой
  «N/M доступны» и значками статуса; время ответа в мс
- **Виджет на рабочий стол:** цвет значка = последний вердикт, тап
  запускает проверку в фоне (приложение не открывается), под значком —
  АБСОЛЮТНОЕ время последней проверки («9:15»), спиннер во время
  проверки, тускнение при данных старше часа
- Кнопка «поделиться вердиктом» справа от чипа сети (текст: вердикт +
  сеть + дата/время + ссылка на RuStore)
- Фоновая проверка (WorkManager, ~15 мин) с уведомлением при смене режима
- Чип «Сеть: …» с пояснениями по ⓘ (мобильный, Wi-Fi, Ethernet, VPN,
  нет сети); VPN определяется, если туннель на самом устройстве
  (VPN на роутере системе не виден — ограничение Android, не баг)
- Светлая и тёмная темы по системной настройке; на Android TV тема
  всегда тёмная (isAppDark: FEATURE_LEANBACK || система)
- Настройка списков сайтов в приложении (шестерёнка): добавление
  по домену, удаление, сброс; хранение в SharedPreferences custom_lists
- Поддержка Android TV: баннер, LEANBACK_LAUNCHER, подсветка фокуса
  пульта (tvFocusHighlight), «Ethernet (кабель)»
- Названия сайтов кликабельны — открывают сайт в браузере
- Ошибки переведены на русский: DNS, таймаут, сброс соединения, TLS;
  расшифровка за «Почему ошибки разные ⓘ»; пометка про Instagram
  видна всегда
- Подвал: копирайт, версия (ссылка на релизы)
- Заготовка удалённого конфига списков (REMOTE_CONFIG_URL, пока пустая)

## Архитектура

Однофайловое приложение: весь код в MainActivity.kt (~1510 строк).
Порядок блоков внутри файла: константы (REPO_RELEASES, RUSTORE_URL) →
data-классы (Probe, ProbeResult, Verdict, ScanState) → ProbeConfig
(встроенные списки) → ProbeStore (пользовательские списки) → Scanner
(сетевые проверки, networkType — VPN проверяется ПЕРВЫМ, вердикт) →
StatusWidgetUpdater + StatusWidget + WidgetScanWorker (виджет) →
CheckWorker + schedule/cancelBackground (фон) → шрифты/цвета/тема
(golosTypography, isAppDark, verdictColors, statusBadgeColors, warnColor,
dangerColor, AppTheme, tvFocusHighlight) → MainActivity → Composable UI
(App, MainScreen, NetworkChip, ShareVerdictButton, GroupCard,
SettingsScreen, EditableGroup, VerdictCard, StatusBadge, ProbeRow,
Footnote, AppFooter).

Состояние: ScanState живёт на уровне App (НЕ внутри MainScreen).
Настройки — SharedPreferences "netstatus": bg_enabled, last_verdict,
last_check_ts, custom_lists.

## Используемые технологии

- Kotlin + Jetpack Compose (Material3), single-activity
- WorkManager 2.9.0, kotlinx-coroutines, HttpURLConnection (без OkHttp)
- Виджет — RemoteViews (Compose в виджете невозможен)
- Шрифт Golos Text (res/font, OFL)
- compileSdk 34, targetSdk 34, minSdk 26, JDK 17, Kotlin 1.9.24,
  AGP 8.4.0, Gradle 8.7 (wrapper генерируется в CI), Compose BOM 2024.06.00
- Сборка: GitHub Actions (ubuntu-latest), release-подпись из секретов;
  локальная сборка не используется

## Важные ограничения (категорически нельзя менять)

1. **applicationId и пакет `ru.netstatus.app`** — идентификатор приложения
   в Android и RuStore. Смена = «другое приложение».
2. **Тема в манифесте только `@android:style/Theme.Material.Light.NoActionBar`.**
   Theme.Material3.* НЕ подключены — ломают сборку. Всё оформление —
   внутри Compose.
3. **Никогда не коммитить keystore.properties и *.jks** — репозиторий
   публичный, первый ключ был скомпрометирован именно так. Пароли —
   только в GitHub Secrets.
4. **Имена артефактов в build.yml** должны совпадать буква в букву
   в шагах Rename и Upload.
5. **gradle.properties с `android.useAndroidX=true` обязателен.**
6. **Иконка** — VectorDrawable «чебурнет» без прямых отсылок
   к Чебурашке (авторские права Союзмультфильма).
7. **Сноска про Instagram** («принадлежит Meta, признанной экстремистской
   и запрещённой в РФ») — юридически обязательная, не удалять,
   не прятать за сворачиваемые блоки.
8. Версия задаётся в app/build.gradle.kts (versionCode/versionName),
   при релизе повышать оба.
9. **material-icons-extended не подключать** (тяжёлый); в базовом наборе
   нет Icons.Filled.Cancel. Icons.Outlined.* базового набора доступны.
10. **VPN-подсказка без сканирования приложений и внешних GeoIP** —
    принципиальная позиция.
11. **ScanState и CoroutineScope сканирования не возвращать внутрь
    MainScreen** — подняты на уровень App намеренно (v0.4.3), иначе
    результаты сбрасываются и проверка обрывается при уходе в настройки.
12. **Подпись виджета — абсолютное время, не относительный счётчик**
    (см. STATE.md, v0.5): RemoteViews не «тикает», MIUI душит фоновые
    обновления — счётчик вечно висел на «сейчас».

## Как нужно работать с проектом

- Пользователь — новичок, git локально не использует. Все правки —
  через веб-интерфейс GitHub (Edit → Ctrl+A → вставить → Commit).
  При создании файлов путь вводить БЕЗ markdown-звёздочек.
- ИИ выдаёт **файлы целиком на замену** (пользователь явно предпочитает
  целые файлы, а не точечные правки), с пошаговыми инструкциями
  на русском.
- Сборка автоматическая на каждый commit; готовый APK — Actions →
  Artifacts (имя WhiteListCheck_v<версия>.apk).
- Релизы вручную: Releases → Draft a new release → тег vX.Y.Z →
  приложить APK → Set as the latest release.
- **Релиз не переигрывать задним числом:** не подменять APK в уже
  опубликованном релизе (versionCode уже занят, магазин отклонит,
  история версий рассинхронится) — выпускать следующую версию.
- Мелкие доработки копить и выпускать пачкой; RuStore частыми
  обновлениями осознанно не бомбим.
- Тестовые устройства: Xiaomi 13T Pro (MIUI/HyperOS, крупный системный
  шрифт — проверять компоновку с ним) и Android TV.
- Язык общения и всех текстов приложения — русский.
- Пользователь внимателен к мелочам дизайна (выравнивания в 4dp
  замечает) — макеты согласовывать до кода.
- От ИИ ждут честной оценки идей, а не поддакивания: если предложение
  сомнительное, сказать прямо и объяснить почему (так отклонили
  двухзонный тап по виджету).

## Известные особенности

- MIUI агрессивно убивает фон: нужны Автозапуск и снятие ограничений
  батареи. Интервал 15 мин — минимум Android.
- ТВ: прокрутка следует за фокусом пульта — нефокусируемые элементы
  ниже последнего фокусируемого остаются за экраном (поэтому копирайт
  в подвале стоит ВЫШЕ ссылки версии). Кнопка сброса списков прилипает
  к нижнему краю — известная косметика, отложено.
- У IconButton внутренние поля 12dp — для выравнивания видимых значков
  по краю контента используются offset-сдвиги (шестерёнка +12dp,
  стрелка назад −12dp, «Почему ошибки разные» −4dp).
- На старых устройствах возможны TLS-ошибки отдельных сайтов
  (устаревшие сертификаты) — пояснение в сноске, не блокировка.
- GitHub-редиректы со старого имени репозитория NetStatus работают.
- При обработке jpg-скриншотов использовать PIL с LOAD_TRUNCATED_IMAGES.
