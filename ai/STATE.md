## В main после v0.5 — СИНХРОНИЗАЦИЯ ВИДЖЕТА И ЭКРАНА (ждёт следующей версии)

Мелкая, но не пустая доработка. Уже закоммичена в main и протестирована
на телефоне — войдёт в следующий релиз (versionCode 13). Отдельно
пересобирать/трогать релиз v0.5 НЕ будем.

- Проблема: виджет и фоновый воркер пишут вердикт в SharedPreferences
  и перерисовывают виджет, но экран приложения (детальные карточки живут
  в памяти ScanState) об этом не знает. Сценарий бага: в приложении
  «всё в норме» → авиарежим → тап по виджету даёт «нет сети» → открываем
  приложение по иконке, а там всё ещё «всё в норме». Экран честно
  показывал устаревший результат, но рядом с виджетом это выглядит
  как враньё.
- Решение: в ScanState добавлено поле `checkedAt` (время получения
  показанного результата; пишется в обеих ветках runScan — и «нет сети»,
  и обычной, одним `now` вместе с last_check_ts). На событие ON_RESUME
  (DisposableEffect + LifecycleEventObserver в MainScreen) сверяем
  `last_check_ts` из prefs с `state.checkedAt`: если снаружи была более
  свежая проверка И вердикт ОТЛИЧАЕТСЯ от показанного — молча вызываем
  runScan(). Совпадающий вердикт не пересканируем (лишний скан ни к чему).
- ВАЖНО: LocalLifecycleOwner берётся из androidx.compose.ui.platform
  (в Compose 1.6.x / BOM 2024.06.00 ещё не deprecated; в 1.7+ переехал
  в androidx.lifecycle.compose — при обновлении BOM поправить импорт).
- Побочный плюс: теперь и после фоновой смены режима открытие приложения
  само подтягивает актуальный результат.

Виджет на рабочем столе (запрос пользователя Рустама; дизайн согласован:

  совместимые view: LinearLayout/ImageView/TextView/ProgressBar);
  res/xml/widget_info.xml (1×1, resizable, updatePeriodMillis 30 мин);
  receiver StatusWidget в манифесте; в MainActivity.kt —
  StatusWidgetUpdater + StatusWidget + WidgetScanWorker + хуки.
- Цвет значка = последний вердикт: норма 66BB6A/4A6B4C, белый список
  EF5350/7C4340, VPN 42A5F5/3E5A78, нет сети 455A64/37474F,
  нейтрально 6D4C41 (не проверялось / UNKNOWN).
- Данные: last_verdict + last_check_ts (пишут ручная, фоновая и
  виджет-проверки). Обновление виджета: после каждой проверки,
  системный тик 30 мин, onResume приложения.
- ПОДПИСЬ ВРЕМЕНИ — АБСОЛЮТНАЯ («9:15» через DateFormat "H:mm"),
  а НЕ относительная «N мин назад». Почему: RemoteViews — статический
  снимок, относительный счётчик сам не «тикает», ему нужна перерисовка
  каждую минуту, а MIUI такие фоновые обновления душит → счётчик вечно
  висел на «сейчас». Абсолютные часы верны без единого обновления.
  До первой проверки — «нажмите».
- Устаревание: >60 мин → setImageAlpha 130 + янтарное время #FFD180.
- ТАП ПО ВИДЖЕТУ = ПРОВЕРКА (не открытие приложения): широковещалка
  самому себе StatusWidget.ACTION_WIDGET_SCAN → onReceive → showChecking()
  (крутящийся ProgressBar на месте подписи; свой drawable в RemoteViews
  анимировать нельзя) + OneTimeWork WidgetScanWorker БЕЗ сетевого
  constraint (иначе при отсутствии сети спиннер крутился бы вечно —
  воркер сам быстро вернёт «нет сети»). WidgetScanWorker уведомлений
  НЕ шлёт (пользователь и так смотрит на виджет) и всегда пишет
  результат+время+обновляет виджет, включая «нет сети».
- ФИКС «открывалось сброшенным»: PendingIntent уведомления теперь
  через packageManager.getLaunchIntentForPackage (ACTION_MAIN/LAUNCHER +
  RESET_TASK_IF_NEEDED) — поднимает существующую задачу как иконка,
  результаты сохраняются. Прямой Intent(MainActivity) этого не давал.
- ГРАБЛИ: при создании файлов через GitHub web путь вводить БЕЗ
  markdown-звёздочек (папка «**layout**» ломала сборку).

## Состав v0.4.3 (что вошло)

- Подъём состояния: ScanState хранится на уровне App (передаётся
  в MainScreen как MutableState, внутри `var state by scanState`) —
  при переходе в «Списки сайтов» MainScreen покидает композицию,
  и remember-состояние внутри него уничтожалось вместе с результатами.
- Подъём scope: CoroutineScope для сканирования тоже уровня App
  (rememberCoroutineScope в MainScreen отменялся при уходе экрана
  из композиции — запущенная проверка обрывалась, running=true
  оставался навсегда → вечное «Проверяю…»). Теперь проверка,
  начатая до ухода в настройки, спокойно доработает в фоне.
- ВАЖНО для будущих правок: state и scope НЕ возвращать внутрь
  MainScreen — оба подъёма намеренные.

## Состав v0.4.2 (что вошло)

- rememberSaveable вместо remember в трёх раскрывашках (чип сети,

  уехавшие за экран элементы вместе с remember — раскрытые ⓘ

## Известные проблемы и особенности

- **VPN на роутере не определяется** — системное ограничение Android
  (для устройства это обычный Wi-Fi), НЕ баг. Вердикт при этом всё
  равно покажет «похоже, включён VPN» по доступности сайтов.
- **ТВ: кнопка сброса списков прилипает к нижнему краю экрана** —
  прокрутка на ТВ следует за фокусом. Косметика; чистого решения без
  экспериментальных API нет. Отложено.
- Версии до 0.3.1 невидимы в лаунчере Android TV — удалять через
  Настройки → Приложения → Показать все приложения.
- MIUI может убивать фоновый воркер без ручных разрешений.
- Интервал фоновой проверки нестрогий (15+ мин).
- На старых устройствах возможны TLS-ошибки отдельных сайтов
  (сертификаты) — пояснение в сноске.
- Инцидент безопасности (закрыт): первый release-ключ скомпрометирован,
  перевыпущен 06.07.2026.
